Service Desk Guidelines

Standard operating procedures for incident management

Reported Incidents

  • Open HelpDesk folder
  • Select correct category and write clear description
  • Follow up daily on incidents transferred to other teams
  • Manager approval required for hardware changes and new access
  • Label CPUs with incident number and seat before repair

Closed Incidents

  • Document full troubleshooting steps before closing
  • If resolved without action: Solved with No Action Taken
  • When closed by other departments, select appropriate transferred area

Escalation Rules

  • Window: Sun-Thu, 08:00-16:00
  • Set status to Awaiting Vendor
  • Include: serial, model, issue, charger status, dates, user details
